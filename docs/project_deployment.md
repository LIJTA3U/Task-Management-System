# Перенос данных

Для переноса данных из локальной PostgreSQL в Docker PostgreSQL использовался механизм резервного копирования и восстановления через pgAdmin.

Этапы:
1. Создание backup локальной базы данных.
2. Развёртывание Docker-контейнера PostgreSQL.
3. Восстановление backup в Docker-базу.
4. Проверка целостности данных и работоспособности приложения.

# Примечание по Docker PostgreSQL

База данных хранится в Docker Volume.
При выполнении команды:

docker compose down -v

все данные PostgreSQL удаляются и создаются заново при следующем запуске контейнеров.

Для сохранения данных рекомендуется использовать резервное копирование через pgAdmin или SQL dump.

# Инструкция по запуску проекта «Система управления задачами»

Описание проекта

Веб-приложение «Система управления задачами» предназначено для регистрации пользователей, создания, редактирования, удаления и отслеживания статуса выполнения задач.

Технологии проекта:

* Frontend: HTML, CSS, JavaScript
* Backend: Node.js, Express.js
* База данных: PostgreSQL
* Контейнеризация: Docker, Docker Compose
* Веб-сервер: Nginx

---

Системные требования

Для запуска проекта необходимо установить:

* Docker Desktop
* Git (опционально)
* Браузер Google Chrome, Microsoft Edge или аналогичный

Операционная система:

* Windows 10/11 (64-bit)
* Linux
* macOS

#Запуск проекта

1. Открыть терминал

Перейти в корневую папку проекта:

cd "Путь_к_проекту/Task Management System"


Пример:

cd "C:\Users\User\Desktop\Task Management System"

---

2. Запустить Docker Desktop

Убедиться, что Docker Desktop находится в состоянии:

Docker Desktop is running

---

3. Собрать и запустить контейнеры

Выполнить команду:

docker compose up --build

При первом запуске Docker автоматически:

* скачает необходимые образы;
* создаст контейнер PostgreSQL;
* создаст контейнер backend;
* создаст контейнер frontend;
* настроит сеть между контейнерами.

---

4. Открыть приложение

После успешного запуска открыть браузер и перейти по адресу:

http://localhost:8080

Backend API доступен по адресу:

http://localhost:3000

---

Остановка проекта

Для остановки контейнеров нажать:

Ctrl + C

или выполнить:

docker compose down

---

Полная очистка контейнеров и базы данных

Для удаления контейнеров и данных PostgreSQL выполнить:

docker compose down -v

Внимание: данная команда удаляет все данные базы данных.

---

Повторная сборка проекта

При изменении кода рекомендуется пересобрать контейнеры:

docker compose down
docker compose up --build

При необходимости полной пересборки без использования кэша:

docker compose build --no-cache
docker compose up

Возможные проблемы и решения

1. Ошибка:

TLS handshake timeout

Решение:

* проверить подключение к интернету;
* перезапустить Docker Desktop;
* выполнить команду:

wsl --shutdown

и снова запустить Docker Desktop.

2. Ошибка:

port is already allocated

Решение:

закрыть приложения, использующие порт 5432 или 3000.

3. Ошибка:

Cannot find module

Решение:

проверить наличие зависимостей в `package.json` и пересобрать контейнеры.
