# Оптимизация и безопасность проекта

1. Выполненные меры безопасности
- Добавлена проверка JWT-токена для защищённых маршрутов.
- Ограничено количество запросов к маршрутам регистрации и входа.
- Подключены безопасные HTTP-заголовки через helmet.
- Удалена зависимость от `user_id`, передаваемого с фронтенда.
- Пользователь определяется на сервере по токену авторизации.

2. Оптимизация базы данных
- Добавлены индексы для ускорения выборок задач и уведомлений.
- Оптимизированы запросы списка задач, статистики и уведомлений.
- Используется сортировка и фильтрация с учётом индексов.

3. Проверка производительности
- Выполнено нагрузочное тестирование backend и frontend.
- Проверена работа API под несколькими одновременными запросами.
- Приложение сохраняет стабильность при базовой нагрузке.

# Вывод
Проект защищён на уровне авторизации, оптимизирован на уровне запросов к базе данных и протестирован под нагрузкой.

# Нагрузочное тестирование

Для тестирования производительности использовалась утилита autocannon.

Результаты:

- Backend API: около 5800 запросов/сек при средней задержке 5 мс.
- Frontend (Nginx): около 4000 запросов/сек при средней задержке 11 мс.
- Защищённый API статистики: около 820 запросов/сек при средней задержке 12 мс.

Приложение сохраняет стабильную работу при одновременном выполнении 10 соединений в течение 15 секунд.
